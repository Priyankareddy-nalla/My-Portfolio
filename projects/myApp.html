MyApp

Project Description

This project involves building a serverless, progressive web application using React and following a Test-Driven Development (TDD) approach. The
application fetches upcoming events using the Google Calendar API and implements several advanced features to enhance user experience and
performance.

Project Features

  • Test-Driven Development (TDD): The app is developed using TDD techniques to ensure high-quality, maintainable code.
  • Google Calendar Integration: Utilizes Google Calendar API and OAuth2 authentication for fetching and displaying upcoming events.
  • Serverless Architecture: Uses AWS Lambda functions for the authorization server, eliminating the need for a traditional server.
  • Cross-Browser Compatibility: Compatible with the latest versions of Chrome, Firefox, Safari, Edge, Opera, and Internet Explorer 11.
  • Responsive Design: The app displays well on all screen sizes, from mobile (320px) to desktop (1920px).
  • PWA Compliance: Passes Lighthouse's PWA checklist for performance, accessibility, and best practices.
  • Offline Functionality: Works offline and in slow network conditions using a service worker.
  • Installable App: Users can install the app on desktops and add it to their home screen on mobile devices.
  • Deployment: The app is deployed on GitHub Pages.
  • Alert System: Implements an object-oriented programming (OOP) approach for an alert system to notify users.
  • Data Visualization: Integrates data visualization to provide insightful information to users.
  • High Test Coverage: Achieves a test coverage rate of 90% or higher.
  • Performance Monitoring: The app is monitored using an online performance tool.

Technologies Used

  • Serverless Backend: AWS Lambda,OAuth2
  • PWA: React for frontend development.
  • Service Workers for offline support.
  • Web App Manifest for add-to-home-screen prompt.
  • Testing: Jest, Puppeteer.
  • Deployed on GitHub Pages.
  • Data Visualization: Recharts.
